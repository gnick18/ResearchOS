# Fix Tasks Not Displaying in Gantt Chart and Sidebar

## Problem Summary
- Gantt chart renders without errors
- Projects display correctly on all pages
- Home page shows tasks within projects correctly (counts and upcoming tasks)
- Tasks are NOT displaying in: Gantt page, DailyTasksSidebar
- No console errors visible
- Tasks existed before the serverless migration edits

## Root Cause Analysis

### Key Insight: Different Filtering Logic

**Home page (WORKS)**:
```typescript
// Groups tasks by project ID - shows all tasks that match a project
const tasks = allTasks.filter((t) => t.project_id === p.id);
```

**Gantt page and DailyTasksSidebar (BROKEN)**:
```typescript
// Filters out tasks if the project doesn't exist in the projects array
const activeTasks = allTasks.filter((t) => {
  const project = projects.find((p) => p.id === t.project_id);
  return project && !project.is_archived;
});
```

### The Problem
If `projects` array is empty or doesn't contain matching projects, tasks are filtered out. This can happen when:

1. **Race condition between queries**: Projects and tasks load independently, and `activeTasks` useMemo runs before projects are loaded
2. **Cache timing issue**: React Query returns cached `projects` from one user but freshly fetched `tasks` from another user

### Why Home Page Works
The home page iterates over `projects` and finds tasks matching each project ID. Even if there's a timing issue, it uses the projects array as the source of truth.

### Why Gantt/Sidebar Fails
They filter `allTasks` based on whether the project exists. If projects haven't loaded yet or there's a user mismatch, all tasks get filtered out.

## Implementation Plan

### Step 1: Add Defensive Check for Empty Projects
When projects array is empty, skip the project existence check temporarily.

**File**: `frontend/src/app/gantt/page.tsx`

```typescript
const activeTasks = useMemo(() => {
  // If projects haven't loaded yet, don't filter by project existence
  if (projects.length === 0) {
    return allTasks;
  }
  
  let tasks = allTasks.filter((t) => {
    if (t.is_shared_with_me) return true;
    const project = projects.find((p) => p.id === t.project_id);
    return project && !project.is_archived;
  });
  // ... rest of logic
}, [allTasks, projects, showShared]);
```

### Step 2: Same Fix for DailyTasksSidebar
**File**: `frontend/src/components/DailyTasksSidebar.tsx`

```typescript
const activeTasks = useMemo(() => {
  // If projects haven't loaded yet, don't filter by project existence
  if (projects.length === 0) {
    return allTasks;
  }
  
  return allTasks.filter((t) => {
    const project = projects.find((p) => p.id === t.project_id);
    return project && !project.is_archived;
  });
}, [allTasks, projects]);
```

### Step 3: Add Username to Query Keys (Long-term Fix)
Ensure all data queries include the current user in the key to prevent cache mixing.

**Files to update**:
- `frontend/src/app/gantt/page.tsx`
- `frontend/src/app/page.tsx`
- `frontend/src/app/experiments/page.tsx`
- `frontend/src/components/DailyTasksSidebar.tsx`

Change from:
```typescript
queryKey: ["tasks"]
queryKey: ["projects"]
```

To:
```typescript
queryKey: ["tasks", currentUser]
queryKey: ["projects", currentUser]
```

And add `enabled: !!currentUser` to prevent queries before user is set.
