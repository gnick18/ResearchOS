# Beta Dev Features Plan

## Overview
Two features for beta mode:
1. **Donation Button** - "Buy me a coffee" link on home page and launch screens
2. **Bug Report Feature** - Easy error reporting via GitHub issues

---

## Feature 1: Donation Button

### Scope
- Show on: Home page (`page.tsx`), `ResearchFolderSetupNew`, `UserLoginScreen`
- Links to PayPal and Zelle (links to be provided during implementation)
- Message: "This is a solo dev project paid for out of pocket. Donations help cover server costs!"

### Implementation

1. **Create `BetaDonationButton` component** (`frontend/src/components/BetaDonationButton.tsx`)
   - Floating button or subtle banner style
   - Opens a modal/popup with:
     - Brief message about solo dev project
     - PayPal link button
     - Zelle info display (phone/email)
   - Configurable via props or config file

2. **Add to pages**
   - `frontend/src/app/page.tsx` - bottom corner floating button
   - `frontend/src/components/ResearchFolderSetupNew.tsx` - subtle link in footer
   - `frontend/src/components/UserLoginScreen.tsx` - subtle link in footer

3. **Configuration** (`frontend/src/lib/config/donation.ts`)
   - PayPal link
   - Zelle info (phone/email)
   - Toggle to enable/disable (for post-beta)

---

## Feature 2: Bug Report Feature

### Scope
- Manual bug reporting via "Report a Bug" button
- Auto-detection of errors with prompt to report
- Creates GitHub issue via pre-filled URL (opens in new tab)
- GitHub repo: `gnick18/ResearchOS`

### Implementation

1. **Create `BugReportModal` component** (`frontend/src/components/BugReportModal.tsx`)
   - Text area for "What were you working on?"
   - Auto-populated error message field (if error detected)
   - "Submit to GitHub" button (opens pre-filled issue URL)
   - "Copy to Clipboard" button as fallback
   - Cancel/Close button

2. **Create global error handler** (`frontend/src/lib/error-reporting.ts`)
   - `window.onerror` handler for uncaught errors
   - `window.onunhandledrejection` for promise rejections
   - Store last error in memory
   - Show toast/modal asking user to report
   - Track if an error was recently caught

3. **Create `ErrorBoundary` component** (`frontend/src/components/ErrorBoundary.tsx`)
   - Wrap app in error boundary to catch React render errors
   - Show fallback UI with option to report bug
   - Allow recovery (retry/render nothing)

4. **Add "Report a Bug" button**
   - Add to AppShell or as floating button
   - Opens BugReportModal without pre-filled error
   - Visible on all pages

5. **GitHub Issue Integration**
   - Generate pre-filled issue URL:
     ```
     https://github.com/gnick18/ResearchOS/issues/new?title=[Bug]%20...&body=...
     ```
   - Include: error message, stack trace (truncated), user description, browser/OS info
   - User must have GitHub account and click "Submit" on the issue page

## UI Design Notes

### Donation Button
- Small, unobtrusive floating button or footer link
- Coffee icon or heart icon
- Modal with warm, friendly tone
- Clear separation of PayPal vs Zelle options

### Bug Report Flow
1. User encounters error OR clicks "Report a Bug"
2. Modal appears with:
   - Title: "Report an Issue"
   - Text area: "What were you trying to do?"
   - Auto-filled error details (if applicable)
   - Browser/OS info auto-detected
3. User clicks "Create GitHub Issue"
4. New tab opens with pre-filled GitHub issue
5. User reviews and submits on GitHub

---

## Edge Cases
- Error while reporting error (fallback to clipboard)
- Very long stack traces (truncate to ~2000 chars)
- No error to report (manual bug report mode)
- User doesn't have GitHub account (note in UI)

---

## Future Considerations
- Remove or hide donation button after beta
- Add serverless function for authenticated GitHub issue creation
- Add telemetry/analytics for error tracking
