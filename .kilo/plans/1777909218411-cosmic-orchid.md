## Executive Summary

Migrate ResearchOS from a FastAPI/Next.js split-stack to a serverless local-first architecture. The application will be hosted as a static website using the File System Access API for direct local file operations and isomorphic-git for GitHub synchronization.

```typescript
import { addDays, isWeekend, nextMonday } from 'date-fns';

export function resolveWeekend(date: Date, weekendActive: boolean): Date {
  if (weekendActive) return date;
  if (isWeekend(date)) return nextMonday(date);
  return date;
}

export function computeEndDate(startDate: Date, durationDays: number, weekendActive: boolean): Date {
  if (durationDays < 1) throw new Error('duration_days must be >= 1');
  
  if (weekendActive) {
    return addDays(startDate, durationDays - 1);
  }
  
  let current = startDate;
  let remaining = durationDays - 1;
  while (remaining > 0) {
    current = addDays(current, 1);
    if (!isWeekend(current)) remaining--;
  }
  return current;
}

export function isWeekendActiveForTask(
  taskWeekendOverride: boolean | null,
  projectWeekendActive: boolean
): boolean {
  return taskWeekendOverride ?? projectWeekendActive;
}
```

## Phase 3: Markdown & Media Resolution

### 3.1 Blob URL Resolver

**New File:** `frontend/src/lib/utils/blob-url-resolver.ts`

```typescript
interface BlobUrlCache {
  [path: string]: string;
}

export class BlobUrlResolver {
  private cache: BlobUrlCache = {};
  private fileService: FileService;
  
  constructor(fileService: FileService) {
    this.fileService = fileService;
  }
  
  async resolveImagePaths(markdown: string): Promise<string> {
    const imageRegex = /!\[([^\]]*)\]\(([^)]+)\)/g;
    const matches = [...markdown.matchAll(imageRegex)];
    
    let result = markdown;
    for (const [full, alt, src] of matches) {
      if (this.isLocalPath(src)) {
        const blobUrl = await this.getBlobUrl(src);
        if (blobUrl) {
          result = result.replace(src, blobUrl);
        }
      }
    }
    return result;
  }
  
  private isLocalPath(src: string): boolean {
    return !src.startsWith('http') && !src.startsWith('data:');
  }
  
  private async getBlobUrl(path: string): Promise<string | null> {
    if (this.cache[path]) return this.cache[path];
    
    const blob = await this.fileService.readFileAsBlob(path);
    if (!blob) return null;
    
    const url = URL.createObjectURL(blob);
    this.cache[path] = url;
    return url;
  }
  
  revokeAll(): void {
    for (const url of Object.values(this.cache)) {
      URL.revokeObjectURL(url);
    }
    this.cache = {};
  }
}
```

- Read from the selected user's directory

## Technical Considerations

### Browser Compatibility
- **Target:** Chromium browsers only (Chrome, Edge, Brave)
- **API:** File System Access API (not supported in Firefox/Safari)
- **Graceful degradation:** Show "unsupported browser" message for non-Chromium

### Data Privacy
- No data sent to external servers
- PAT stored locally only
- All processing client-side

### File System Access API Limitations
- Requires user gesture to request permission
- Handle may be invalidated on browser restart (re-prompt needed)
- No direct path access - must traverse from handle

### Performance
- Cache blob URLs to avoid repeated file reads
- Batch file operations where possible
- Use Web Workers for heavy computations (shift engine)

### Error Handling
- Permission denied scenarios
- Invalid/corrupted JSON files
- Git merge conflicts
- Offline operation (no GitHub sync)
