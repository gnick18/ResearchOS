# Improve Opening Screen for New Users

## Problem
The current opening screen only shows a single "Select Folder" button, which is confusing for new users. They don't understand:
1. They can link to an existing ResearchOS folder (if they have data from another device)
2. They can create a new folder by selecting an empty folder and initializing it

## Current Flow
1. Initial screen: "Connect Research Folder" with single "Select Folder" button
2. After folder selection:
   - Valid ResearchOS folder (has `users/` dir) → User selection screen
   - Invalid/empty folder → "Initialize New Folder" prompt
   - User cancels picker → Returns to initial screen

## Proposed Changes

### New Initial Screen Layout
Replace the single button with two clear options:

**Option 1: Link Existing Folder**
- For users who already have a ResearchOS folder (e.g., synced via OneDrive/iCloud)
- Description: "Connect to an existing ResearchOS folder with your projects and data"
- Button: "Link Existing Folder"

**Option 2: Create New Folder**
- For new users starting fresh
- Description: "Create a new ResearchOS folder. First create an empty folder on your computer, then select it below."
- Button: "Select New Folder Location"
- Helper text: "Tip: Create a folder called 'ResearchOS' in your Documents or cloud sync folder first"

### Implementation Steps

1. **Add state for expandable help** in `ResearchFolderSetupNew.tsx`:
   - `const [showHelp, setShowHelp] = useState(false)`

2. **Replace the initial screen (lines 283-371)** with two-card layout:
   - Keep returning user reconnect screen as-is (when `lastConnectedFolder` exists)
   - For new users, show two side-by-side option cards

3. **Create the two option cards**:
   - Left card: "Link Existing Folder" with folder icon
   - Right card: "Create New Folder" with sparkle icon + expandable help

4. **Both buttons call `connect()`** - the same function, but with different user expectations

5. **Style updates**:
   - Use existing gradient background and card styling
   - Cards should have hover effects and consistent spacing
   - Help section uses a nested box with subtle background

### User Experience Choices
- **Returning users**: Show simplified "Reconnect to [folder name]" screen (keep current behavior)
- **New folder help**: Add expandable help section with step-by-step instructions

### User Flow After Changes

**Scenario A: Existing ResearchOS User**
1. Sees two options, clicks "Link Existing Folder"
2. Selects their existing ResearchOS folder
3. Folder validates successfully → User selection screen

**Scenario B: New User**
1. Sees two options, reads "Create New" instructions
2. Creates empty folder on their computer (e.g., `Documents/ResearchOS`)
3. Clicks "Select Location"
4. Selects the empty folder they created
5. App shows "Initialize New Folder" prompt
6. Clicks "Initialize Folder" → Creates folder structure
7. Creates their first account → Enters the app

### Edge Cases Handled
- User selects wrong folder type (e.g., new user selects existing folder) → Works fine, they just see user selection
- User selects wrong folder type (e.g., existing user selects empty folder) → Works fine, they get initialize prompt and can go back
- User cancels picker → Returns to initial screen with both options
- Returning users (lastConnectedFolder exists) → Show simplified "Reconnect" screen (single button, current behavior preserved)
