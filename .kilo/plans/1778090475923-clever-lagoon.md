# Fix: User Task/Goal Isolation in Local Mode

## Problem
When viewing different user profiles in Lab Mode or switching users, all tasks and goals from GrantNickles are displayed for every user. This is because the `JsonStore` class in `local-api.ts` always reads from the **currently cached user's directory**, rather than from each specific user's directory.

## Root Cause Analysis

### Current Behavior
1. `JsonStore.getBasePath()` uses `getCurrentUserCached()` which returns a cached user value
2. `labApi.getTasks()`, `labApi.getProjects()`, etc. all call `tasksStore.listAll()` / `projectsStore.listAll()`
3. These methods always read from `users/{cachedUser}/tasks` regardless of which user's data should be displayed

### Expected Behavior
- Lab Mode: Should aggregate data from ALL user directories (`users/{username}/tasks` for each user)
- User Switch: Should read from the selected user's directory after switching
- Goals: Should follow the same user-specific pattern as tasks

## Solution

### Implementation Details

#### Step 1: Extend `JsonStore` class (`frontend/src/lib/storage/json-store.ts`)

Add private method and public methods for user-specific operations:

```typescript
private async getBasePathForUser(username: string): Promise<string> {
  return `users/${username}`;
}

async listAllForUser(username: string): Promise<T[]> {
  const basePath = await this.getBasePathForUser(username);
  const dirPath = `${basePath}/${this.entityName}`;
  const fileNames = await fileService.listFiles(dirPath);
  const records: T[] = [];
  for (const fileName of fileNames) {
    if (!fileName.endsWith(".json")) continue;
    const record = await fileService.readJson<T>(`${dirPath}/${fileName}`);
    if (record) records.push(record);
  }
  return records.sort((a, b) => a.id - b.id);
}
```

#### Step 2: Update `labApi` methods (`frontend/src/lib/local-api.ts`)

Import `discoverUsers` from user-discovery:
```typescript
import { discoverUsers } from "./file-system/user-discovery";
```

Update `labApi.getTasks()`:
```typescript
getTasks: async (params?: { exclude_goals?: boolean; usernames?: string }): Promise<LabTask[]> => {
  const allUsers = await discoverUsers();
  const tasks: LabTask[] = [];
  
  for (const username of allUsers) {
    const userTasks = await tasksStore.listAllForUser(username);
    for (const t of userTasks) {
      const task = computeTaskEndDate(t);
      tasks.push({
        id: task.id,
        name: task.name,
        // ... other fields
        username: task.owner || username,
        // ...
      });
    }
  }
  
  return tasks;
}
```

Similarly update:
- `getProjects()`: Use `projectsStore.listAllForUser(username)`
- `getExperiments()`: Same pattern
- `getPurchases()`: Same pattern
- `getUserTasks(username)`: Use `tasksStore.listAllForUser(username)` directly
- `getUserProjects(username)`: Use `projectsStore.listAllForUser(username)` directly

#### Step 3: Update `labApi.getUsers()` for color support

Read user colors from `_user_metadata.json` if it exists, or assign default colors.
