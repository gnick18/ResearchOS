# Fix: User Data Not Displaying After User Switch

## Problem
When switching between users, the app correctly shows the user list and allows clicking between users, but the data (notes, projects, tasks, etc.) from the selected user's folder is not displayed. 

**Root causes**:
1. `json-store.ts` caches the current user in a module-level variable (`currentUserCache`)
2. When `setCurrentUser()` is called, it updates IndexedDB but does NOT clear this cache
3. All `JsonStore` operations continue using the stale cached username, reading from the wrong user's folder
4. React Query caches have query keys like `["notes"]`, `["projects"]` that don't include the current user
5. When user switches, the cached query data is still returned (stale data from old user)

## Solution

### Step 1: Clear the user cache when switching users
**File**: `frontend/src/lib/file-system/file-system-context.tsx`

Import `clearCurrentUserCache` and call it in `setCurrentUser`:

```typescript
import { clearCurrentUserCache } from "../storage/json-store";

const setCurrentUser = useCallback(async (username: string) => {
  clearCurrentUserCache();  // Clear cache FIRST
  await storeCurrentUser(username);
  setState((prev) => ({ ...prev, currentUser: username }));
}, []);
```

### Step 2: Invalidate React Query cache on user switch
**File**: `frontend/src/lib/providers.tsx`

The `queryClient` is inside `AppContent`, so add a `useEffect` that invalidates queries when `currentUser` changes:

```typescript
useEffect(() => {
  if (currentUser) {
    queryClient.invalidateQueries();
  }
}, [currentUser, queryClient]);
```

Add this after the existing `useEffect` block (around line 35).

## Files to Modify

1. `frontend/src/lib/file-system/file-system-context.tsx` - Import and call `clearCurrentUserCache` in `setCurrentUser`
2. `frontend/src/lib/providers.tsx` - Add useEffect to invalidate queries when currentUser changes
