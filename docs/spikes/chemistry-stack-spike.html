Chemistry Workbench stack spike

Live proof that the recommended browser-only stack actually runs, plus the measured WASM footprint. Ketcher canvas + RDKit.js engine + PubChem import, no backend.

1Measured footprint acceptable, lazy-loaded

Real asset sizes from the published packages (jsDelivr, 2026-06-10). None of this is in the core app bundle. It loads only when a user first opens the chemistry feature, then browser-caches.

AssetRoleRawGzip

Read: ~5.7 MB gzipped on the first time the editor opens, then cached. For a desktop research tool this is fine because it is behind a dynamic(ssr:false) import and never touches the home/dashboard load. The lighter alternative (OpenChemLib, ~1.5 MB, editor+engine in one) trades a best-in-class editor and RDKit's stronger substructure search for the smaller size. The footprint alone does not force that downgrade.

2RDKit.js engine, running live LIVE WASM

This is the load-bearing engine (and the one with the maintenance-transition caveat). It is loaded from CDN right now in your browser. Ketcher emits a SMILES or Molfile; RDKit consumes the exact same contract, so this proves the engine half end to end. Type a SMILES or paste a Molfile.
