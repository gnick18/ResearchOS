# Common-Features Detector: Feature-Database Licensing Research

Status: research complete 2026-06-03 (deep-research, 21 sources, 25 claims verified, cited). Input for the deferred "Detect Common Features" auto-annotation feature.

Question: is there a publicly available, openly-licensed, REDISTRIBUTABLE feature-sequence
database we can bundle offline (client-side, AGPLv3) for a SnapGene-style common-features
detector?

## Bottom line

There is NO single turnkey database we can drop in. But there ARE cleanly-licensed building
blocks to assemble a curated starter set from. The detector is buildable; it needs a bounded,
SOURCED (never fabricated) data-curation pass first.

## Usable (clean, verified licenses)

- FPbase (fluorescent proteins). The DATA is copyright-free (CC0-style public-domain waiver;
  attribution requested). USABLE: bundle the raw FP sequences (NOT the CC-BY-SA site
  compilation, NOT the GPL code). Source: fpbase.org/terms, fpbasepy API.
- UniProt / Swiss-Prot (protein features: resistance markers, epitope tags, etc.). CC BY 4.0
  (attribution only; no copyleft conflict, fine as an aggregate alongside AGPLv3 code).
  USABLE with an attribution notice. Source: uniprot.org/help/license.
- Rfam (regulatory ncRNA) and Pfam / InterPro (protein families). CC0 public domain. USABLE
  freely. Caveat: these cover RNA/protein families, NOT plasmid markers/origins/MCS, so they
  only contribute regulatory-RNA + protein-family elements.

## Avoid (license problems)

- pLannotate bundled DB. Code is GPLv3, but the feature DB is a 4-source composite
  (Rfam + FPbase + Swiss-Prot + a SnapGene-derived "GenoLIB" set, ~13,240 features). The
  SnapGene-derived component has NO stated data license -> redistribution risk -> do NOT copy
  pLannotate's DB wholesale. (We can reuse its CLEAN upstreams ourselves, just not its
  SnapGene-derived part.)
- iGEM Registry of Standard Biological Parts. Bulk April-2024 mysqldump exists, but content is
  CC-BY-SA (ShareAlike) -> copyleft hazard for AGPLv3 bundling. A proposed drop to plain CC-BY
  was NOT confirmed ratified. Flagged risk; do not bundle without direct license confirmation.
  (Also the live page 403'd; in read-only/archive mode.)
- SnapGene Common Features (.ftr). Proprietary. UNUSABLE. (This is exactly what contaminates
  pLannotate's GenoLIB component.)

## The gap

The plasmid-SPECIFIC categories the detector most needs (origins of replication, exact
MCS/cloning sites, common terminators, some promoters) are NOT cleanly covered by
FPbase/UniProt/Rfam as turnkey sets. Resistance markers + tags can come from Swiss-Prot;
FPs from FPbase; regulatory RNA from Rfam. Origins / MCS / terminators / common promoters
likely need HAND-CURATION from public-domain primary records (NCBI GenBank entries are public
domain), each entry traceable to a source, explicitly AVOIDING any SnapGene-derived data.

## Recommendation

Build a curated, license-tagged STARTER feature set by PULLING sequences from the clean
sources (FPbase data, Swiss-Prot CC BY, Rfam/Pfam CC0, NCBI public-domain records) via a
fetch-and-format pipeline. CRITICAL: sequences must be FETCHED + verified from primary
sources, never hand-typed/generated by a model (hallucination risk). Each entry carries its
source + license tag; ship an attribution file (FPbase, UniProt). Format: FASTA + JSON sidecar
metadata, small enough for client-side; the existing pairwise aligner then aligns these
references against the user's sequence to propose annotations.

Minimal-viable starter subset (the ~most-common features to ship first, per-feature sourcing
to be mapped during curation): resistance markers (AmpR/KanR/CmR), origins (pUC/ColE1/pBR322/
p15A), promoters (T7/lac/CMV/CAG), terminators (T7/rrnB), epitope tags (His6/FLAG/HA/Myc),
common fluorescent proteins (from FPbase), and a common MCS.

## Open items before build

- Map each starter feature to a specific clean source + verified sequence + license tag.
- Confirm the assembled set's size is small enough to ship client-side (expected: easily, a
  few hundred short features as FASTA/JSON).
- Attribution file for CC-BY sources (UniProt) + requested FPbase attribution.
