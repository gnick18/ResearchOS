## How orchestration works on this repo

You are the master bot. There is **no sub-manager tier** (retired
2026-05-23). When work needs doing:

1. **Trivial inline work** (1-2 line tweak, copy fix, single test
   update): do it directly with Edit/Bash, commit, sync the session
   anchor.

2. **Anything scoped, non-trivial, or multi-file**: dispatch a sub-bot
   via the `Agent` tool with `isolation: "worktree"`. Brief them with:
   - Full context (the bot has zero memory of this conversation)
   - Files to investigate / touch
   - Explicit voice rules (no emojis, no em-dashes, "BeakerBot" not
     "beakerbot")
   - "Commit early and often" for wifi-watchdog resilience (the
     watchdog kills agents after 600s of no progress; a wifi drop
     during the working window can lose uncommitted work)
   - "Use `npx vitest run ...` from `frontend/`, NOT `pnpm test`"
     (pnpm hits a build-script consent prompt on this machine)
   - Sign-off line for the report

3. **Cherry-pick the sub-bot's commits to `main` on report.** Conflicts
   are usually small (other in-flight chips touching the same registry
   files). Take "theirs" when a chip supersedes an earlier inline fix.

4. **Sync the session anchor** after cherry-picks: `git branch -f
   _main_snapshot main`. Sub-bot worktrees branch from the session
   anchor; a stale anchor causes anchor-drift conflicts on later
   cherry-picks.
