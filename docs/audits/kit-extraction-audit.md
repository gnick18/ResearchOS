## How to read this

For every catalog template that bundles a vendor source PDF, this checks whether the CRITICAL values we extracted appear literally in the vendor PDF's text (deterministic string matching, no LLM, no network).

- A value listed under **Review (structured)** is the higher-signal flag: it was read from a typed payload field (a temperature, time, m/z, column spec, reagent name) and was NOT found in the PDF text.
- A value listed under **Review (free-text)** was mined from our own prose (card description / notes / markdown body). A miss there is usually just paraphrase, not drift.
- **NOT FOUND is a REVIEW FLAG, not proof of error.** PDF tables mangle whitespace, vendors phrase values differently, some values are figures or deliberate authored defaults, and unit conversions won't match literally. Confirm against the actual PDF before changing anything.
