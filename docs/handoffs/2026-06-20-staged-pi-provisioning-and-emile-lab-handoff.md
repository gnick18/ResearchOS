## COORDINATION + GOTCHAS

- THE PEPPER IS A SENSITIVE VERCEL VAR. `vercel env pull` returns `DIRECTORY_HMAC_PEPPER` BLANK by design (good security). `DATABASE_URL` pulls fine. So you CANNOT hash an email under the prod pepper outside the running server. To learn an email's current-pepper hash, stage it (the stage route computes it) and read the staged `pi_email_hash`, or observe behavior. Always delete any pulled `.env` immediately.
- A sub-bot can stall mid-task. The unstage build bot finished the db helpers then stopped before the routes/UI/tests; its work was UNCOMMITTED in its worktree. There is no resume-a-subagent tool in this harness, so the orchestrator finished it inline (read the worktree diff, wrote the rest, committed on the branch, merged). If a bot reports a fragment, check its branch + worktree for uncommitted work before re-dispatching.
- MULTI-AGENT MAIN moves fast. Build in isolated worktrees, symlink node_modules for the tsc/vitest gate, commit on a branch, then merge + verify the MERGED tree (rm -rf frontend/.next/types first, tsc standalone, run the touched suites). All four features were verified this way.
- The lab-domains lane owns `proxy.ts` + lab-site routing. The slug-rename page-level redirect does not touch it; the edge-redirect follow-up is flagged for them.
