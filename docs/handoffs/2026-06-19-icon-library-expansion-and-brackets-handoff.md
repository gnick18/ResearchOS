## 1. Open-asset federation (the main lane)

The library is an `AssetSource` federation: each source is a standalone ingest
adapter on the shared `frontend/scripts/asset-ingest/lib.mjs` seam (normalized
Asset + license policy + per-source credit formatter + per-fill-preserving SVG
sanitizer). Adding a source is one adapter, zero app change. The app reads the live
CDN manifest from `assets.research-os.com`.

**Before this session:** PhyloPic (11,738) + BioIcons (2,821) = 14,559. Heavily bio.

**Added this session (5 sources):**
- `ingest-reactome.mjs` — Reactome Icon Library, CC-BY 4.0, 2,569 molecular /
  cellular / pathway icons from `reactome/reactome_illustrations` GitHub, joined to
  ContentService for per-icon designer attribution. Mostly multi-fill.
- `ingest-healthicons.mjs` — Health Icons, MIT, 1,482 medical/public-health glyphs
  (filled + outline) from `resolvetosavelives/healthicons`. Single-fill.
- `ingest-tabler.mjs` — Tabler Icons, MIT. Ingests ONLY the science/tech categories
  (1,006 of 5,093) via `tablerCategory()` and skips the UI/brand flood
  (System/Arrows/Brand/E-commerce/...). Carries Tabler's rich per-icon tags for
  search. NOTE outline icons are stroke-based, so they display fine but do not
  single-tint via the fill-recolor path (a recolor gap, not a display issue).
- `ingest-devicon.mjs` — Devicon, MIT, 578 language/framework/tool logos for CS
  diagrams. TRADEMARK GUARDRAIL: keeps brand colours, tags `logo`/`brand`, sets
  `isLogo: true` so the UI can keep recolor off, and the credit names the trademark.
- `ingest-scidraw.mjs` — SciDraw, CC-BY, ~631 figure-quality physics/apparatus/neuro
  drawings. Multi-path SVG so per-fill recolor works well. Built from Grant's Chrome
  devtools capture: enumerate `/?page=N`, scrape each `/drawing/<id>` for
  title/category/authors/DOI, download `GET /download/<id>` with 503-retry+backoff
  (the endpoint cold-starts). License-override guard is scoped to the drawing's own
  info block (the footer's "unless stated otherwise" is the site default, NOT a
  per-drawing override).

**Vetting bar applied to every source** (the federation invariant): true SVG,
commercial + derivative open license (CC0/PD/CC-BY/CC-BY-SA/MIT/BSD), machine
readable per asset, redistributable, no NC/ND. Excluded: Simple Icons (brand /
trademark, low figure value), BioRender / Flaticon / Noun Project (license).

## 2. Mechanics worth remembering (these bit during the session)

- **Append, never `rclone sync`.** The full corpus is regenerated by seeding a LOCAL
  copy of the LIVE manifest (`cp live-manifest.json out/bundle/manifest.json`) then
  running adapters to append. Sync to R2 with `rclone copy` / `copyto` ONLY (sync
  deletes the 14k existing SVGs that are not in the local bundle).
- **Category mapping onto the LOCKED taxonomy.** The grouped sidebar matches exact
  curated leaf names, so each source's raw category strings are mapped onto existing
  leaves by pure tested mappers in lib.mjs (`reactomeCategory`, `healthiconsCategory`,
  `tablerCategory` which also doubles as the science/tech allowlist, `scidrawCategory`
  keyword map). The raw word is kept as a search tag. Verified 0 new assets pool into
  "Other".
- **Embeddings (semantic search).** `embed-assets.mjs` was migrated to
  `@huggingface/transformers` v3 (matches the client) + `dtype:"q8"`
  (onnx/model_quantized.onnx = the same weights the browser loads) + loads the model
  FROM R2 via `env.remoteHost` because HuggingFace downloads time out in this
  environment. Row order == manifest order; `semanticSearch` guards
  `manifest[h.row]` so the upload order can never crash a fresh client. Regenerate +
  re-sync the `.bin` + `.meta.json` WITH the manifest every time the corpus changes
  (count mismatch is the only way to break it).
- **Cloudflare cache.** Right after an upload one edge node served a stale
  embeddings bin once, then self-resolved. The assets are `cf-cache-status: DYNAMIC`
  (uncached), so it is transient. If smart search ever throws "embeddings too short",
  hard-reload once.

## 5. Data Hub significance-bracket auto-spacing (the visual fix)

`frontend/src/lib/datahub/plot-spec.ts`, commits `a473ee642` + `9241af5af`.

The original report was brackets overlapping. A first attempt (bump the tier step
18 -> 25) made it WORSE: bigger tiers need more axis inflation, so a dense 4-group
all-pairs 6-tier stack at the default 430x340 panel either inflated the axis to
~0-14 (data crushed into the bottom quarter) or, a hair shorter, bailed the headroom
reservation and overlapped the data. This was the "larger auto-detection error"
Grant suspected.

**Real fix, two coupled parts:**
1. Cap the headroom band at 52% so the DATA always keeps ~half the plot (no more
   axis blow-up).
2. Lay the stack out at the comfortable tier, and if the top would clip the ceiling,
   COMPRESS the tier step and shrink the star font (new `BracketGeometry.labelFont`)
   to fit the reserved band, instead of shoving the stack down into the data.
   Then widened the lowest-bracket-to-data air (gap 16 -> 24).

Verified by headless rasterize (write SVG to a temp file, `qlmanage -t`, read the
PNG) on the worst cases: 5-group / 10-bracket, 6-group / 15-bracket, mixed
`*`/`**`/`***`/`ns` with angled labels, uneven heights + SD error bars. All clean,
data readable, no overlap. 100/100 plot-spec tests, plus a new dense-case regression
asserting both invariants (data >= 40% of axis, label clears the bracket above).
