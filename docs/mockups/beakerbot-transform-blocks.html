Each block is a real Data Hub transform (the same 5 ops a user can run by hand from the table page, normalize / transpose / per-cell function / remove baseline / fraction of total), shown in the SAME block language as the manual TransformDialog, name, plain-language blurb, the params, and a LIVE preview computed by the real engine (never a guess). You approve each step (or edit its params), then Create builds one derived table with the recipe attached. Today the engine does single ops, so v1 is one block per table; this stacks into a multi-step pipeline the moment the Data Hub agent lands the recipe generalization, same UI. The result opens as an inline embed in the chat.
