Data Hub, transform palette readability

2026-06-13. The op palette reads as a sea of text. Three redesigns. Mark up, approve or flag each, Export.

What you are reviewing

The transform builder's left palette lists every operation under category headings (Filter & select, Edit values, Missing data, Strings, Compute, Type & schema, Reshape & summarize). Today the headings are tiny and muted, so they read at nearly the same weight as the clickable ops below them, and 30+ items in one scroll feels like a wall. Below: the current build, then three ways to fix the hierarchy. Each option is independently approvable so you can mix and match (for example, take A's category icons with B's collapsing).

Note on icons: the glyphs here are sketches. The real build uses only registry glyphs via the Icon component (one glyph per meaning), so the category set would be drawn from existing icons or flagged for your sign-off before build.

Current build
Before, the flat wall

Headings are 10px muted uppercase with 2px of separation, the same left edge as the ops. Nothing visually says "this is a label, those are buttons." Scanning to the group you want means reading most of the list.

A
Lowest lift
Headings that look like headings

Each category gets a hairline rule above it, real breathing room, a category glyph, and a distinct color and weight (purple, the accent we use for section tags). Ops indent slightly so the eye reads header then items. Pure CSS, no behavior change, no scroll change.

  • Fastest to ship, lowest risk.
  • Still one long scroll, just far easier to scan.
B
Kills the wall
Collapsible categories with counts

Each category is a collapsible header with an icon, a count badge, and a chevron. Common groups open by default, the rest collapsed, so the palette is short and you expand what you need. Typing in the search (top) auto-expands matching groups. This is how Notion, Linear, and most modern command palettes handle a long verb list.

  • The 30-item wall becomes 7 lines until you open a group.
  • One extra click to reach a collapsed op (mitigated by search and remembered open-state).
C
Most polished
Search-first with tinted category cards

A prominent search box leads (filter ops by name as you type), and each category is a soft card with a tinted header bar so groups are visually chunked, not just spaced. Most scannable and most "designed," slightly more vertical space per group.

  • Search-first matches how people actually look for a known op.
  • Card chrome costs a little height per group versus A.
