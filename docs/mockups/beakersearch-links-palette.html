BeakerSearch on the Links page, before and after

Empty-query Cmd-K with one of your links selected. Mark each section below.

Before (today)

The Links page registers no source, so Cmd-K shows only the always-on global layer. No board context, no per-link actions, no open-the-bookmark, no jump-to.

After (proposed Links source)

A context card (board scope + the selected link), per-link actions gated by ownership (edit / category / visibility / delete on your own; open + copy on shared-in), jump to a card, and the new external-open kind, open a bookmark or a typed url in a new tab.

What the Links source adds. Mark each Approve or Disapprove.

All actions wire to the page's real handlers (per docs/proposals/beakersearch-links.md). Writes gate on isOwnLink, shared-in links are view-only (open / copy only). External opens always use a new tab with noopener and never auto-open. No new chip color, links use a neutral blue link chip + a link icon.
