User-customizable navigation bar

Design review. A slim bar where each person drags tabs between the bar and a Hidden tray, with no extra button cluttering the bar.

This is a design mockup, not the real app. It prototypes the slimmed nav bar you approved (six tabs inline, the rest behind More) plus the part you asked for next. Which tabs sit inline versus hidden is now yours to set, by dragging them like app icons. There is a sensible default, and you reshape it for how you actually work. The whole interaction below is real, so you can feel the long-press into edit, the jiggle, and dragging tabs in and out of Hidden before we build it.

Enter edit mode three ways 1. Long-press any tab (hold about half a second). 2. Right-click a tab or empty bar space, then Customize tabs. 3. Open More, then Customize tabs at the bottom. While editing drag tabs to reorder, drag one into Hidden to hide it, drag one out of Hidden back onto the bar to show it. Press Enter or Esc, or the Done pill, to finish. Reset to default lives inside edit mode. Comment mode click any element to pin a note and tag it Approve or Disapprove. Export Copy or Download .md, then paste it back into chat.

The slim bar, yours to arrange Live Drag to customize

Six tabs inline by default (Home, Workbench, GANTT, Methods, Data Hub, Calendar), the other five behind More (Sequences, Chemistry, Inventory, Purchases, Links). Methods is the active tab. The bar is about 40px tall so the page below reclaims the space. Try a long-press on a tab to start, or right-click empty space. Your arrangement saves to this browser, so a reload keeps it.
