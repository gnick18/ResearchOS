A workflow macro is a saved, named sequence of BeakerBot steps you replay with one command. It is the user-authored cousin of the plan card. Locked decisions, deterministic replay (the saved steps run in order under one approval, destructive steps still self-confirm), record-first authoring plus a hand editor, and personal macros now with lab-shared macros built into the shape for later. These five sections are the surfaces. Approve or flag each.

After any run, a "Save these steps as a macro" affordance appears on the steps panel. It captures the deterministic action and data tools that ran, in order, with their exact arguments. The navigation clicks between them are dropped so the macro is the meaningful steps. Clicking it opens the editor (section 4) pre-filled to name and trim before saving.

Macros appear in the same / menu as the six curated commands, in a separate "Your macros" group, tinted violet with a "macro" tag so a saved routine reads differently from a one-shot command. A curated command prefills the composer for the model to interpret. A macro is already fixed, so selecting it stages a run (section 3) instead of typing prose.

Selecting a macro raises one Run card showing every step (left). On Run, the steps replay in order without re-asking (right). The one carve-out is the safety rule the plan card already enforces, a destructive or outward step (send, share, delete, pay) pops its own confirm at the moment it runs, even inside an approved macro. Skip drops just that step and continues. A failed step stops the run and says which one.

The same sheet serves capture, edit, and author-from-scratch (empty step list plus Add step). Each step shows its underlying tool, can be reordered, relabeled, or removed. An argument that froze a date at record time gets a "fixed date" marker so the freeze is visible and changeable, this is the honest face of deterministic replay. Smart relative dates that re-resolve each run are a planned fast-follow, not v1.
