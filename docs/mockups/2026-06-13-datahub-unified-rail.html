Data Hub rail on the unified shell

Before / after. Same family-tree, gains resize + collapse + filter, matches the other pages.

Data Hub already has a good rail, so this is not a rebuild. It is the one rail of the four data pages that is a fixed 232px, cannot collapse, and has no type-to-filter. Data Hub is also where the widest tables live, so a rail that can get out of the way matters most here. The proposal keeps the signature family-tree (folders, then tables, with each table's Results and Graphs nested under it) exactly as-is, and puts it on the same shell Sequences, Chemistry, and Tree Studio use.
