2. Revert-to-here confirm

click to comment

Revert to this message?

The two messages after this point will be removed. This cannot be undone.

Clicking Revert to here on any user message shows this confirm before discarding later turns. The confirm states exactly how many messages will go. Design question for Grant: should the removed turns be silently discarded, or should they be preserved as a "branch" the user can navigate back to? Branching is friendlier but adds UI complexity to the rail.

The status line is a single compact bar: BeakerBot glyph, elapsed time, token count, number of active tool calls (pulsing sky dot), and a human-readable phase word ("working", "almost done", "wrapping up"). A "details" link toggles the expanded step list below it. The Stop button (already shipping) stays visible at all times during a run. Segments use a middot separator, no colons.

When a turn finishes, the status line stays visible for the rest of the conversation so you can see what each turn cost. Token count is real (engine-reported).
