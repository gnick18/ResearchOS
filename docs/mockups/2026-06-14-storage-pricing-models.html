Storage pricing: storage-only vs storage + activity

Interactive model for ResearchOS cloud storage. Constants from lib/pricing/assumptions.ts (FLAGGED placeholders). 2026-06-14. Beta is free; this models the flip.

Bottom line: storage-only pricing loses money only on low-storage / high-activity users (heavy real-time collab or automation), and the worst case is a free user hammering collab. It is fully bounded by the activity throttle: set each tier's write ceiling so ceiling × $1.50/M ≤ price and you never lose money. For typical users, activity is pennies, so storage-only just trims margin. The throttle is the load-bearing guard, not optional.
