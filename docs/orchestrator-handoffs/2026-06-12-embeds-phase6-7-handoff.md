## Identity stability (the rename worry, verified in code, for the next session's context)

Grant worried renames would break embeds / forks / shares / collab. Verified the architecture is sound:
- **Objects are ID-keyed and immutable.** `JsonStore` stores `{id}.json` (id from `_counters.json`); the display name is a field, not the filename. Renaming is a field edit. Embeds carry the id in the URL, forks clone to new ids, task->project is `project_id`, deps are by id. All survive a rename.
- **Cross-user is keyed on immutable identity.** Content identity `portableIdentityFor` (source_uuid / InChIKey / sequence fingerprint); user identity is a crypto key FINGERPRINT (`lib/sharing/identity/device-key.ts` x25519/ed25519), NOT the username; collab is keyed by `docId` (`lib/collab/server/auth.ts`). A recipient renaming their copy does not touch the host; real-time collab renames sync as ordinary edits.
- **The one name-as-key is USERNAME** (`users/<username>/` paths + `<owner>:<id>` composite keys), but it has dedicated migration handling (`renameUserMetadataEntry`, `lib/user-rename.test.ts` 200+ lines, the per-user migration runner).

## Gotchas / lessons (READ before integrating on the shared main)
1. **The shared main checkout is a treadmill and it bit ~4 times this session.** Two distinct failure modes: (a) a concurrent session's `git commit` ran in the window between my `git merge --no-commit --no-ff` and my `git commit`, ABSORBING my staged merge into THEIR commit (the P7-1 merge landed labeled `86a4e04f0 docs(beakersearch)...` with `28570a35a` as its second parent, intact but mislabeled). (b) Stale-anchor PHANTOM deletions: `git diff main..branch` showed huge deletions in unrelated files (datahub `engine/fit/global.ts` -428, `SequenceEditView.tsx` -453) that were just main's NEWER work the branch lacked, NOT real deletions. THE SAFETY THAT WORKED EVERY TIME: `git merge --no-commit --no-ff <branch>`, then `git diff --cached --stat` and explicitly grep the staged file list for any foreign bleed; only commit if the staged set is exactly your files. Disjoint-file merges resolve correctly (main's work is kept), the phantom diff is harmless. ALWAYS `git branch --show-current` == main and check for a foreign `.git/MERGE_HEAD` before starting.
2. **Sub-bot worktrees anchor on a STALE base.** Every bot this session had to `git merge main --no-edit` in its worktree first; even so, main advanced further before integration. Re-verify gates on the MERGED tree (the `--no-commit` working tree), not on the bot's report alone. Verify bot work yourself (read the diff, re-run the suite).
3. **Data-shape that is inert until used can merge to main.** P7-1's `<note>.ros-embeds.json` sidecar is only written on an explicit Pin click (no migration, no auto-mutation). Merging it to main is safe AND is the only way Grant can verify on his :3000. The "data-shape waits for verification" rule is really about migrations / auto-mutations; an inert opt-in feature is fine to land flagged-for-verify. P7-2's normalize-on-save is similar (only fires on `![[` content).
4. **CM6 view-only invariant.** `embed-widget.ts` was deliberately view-only to hold the byte round-trip. P7-3 and P7-1 break it ONLY in response to an explicit user action (view-switch, pin), via a fragment-only rewrite that preserves the rest of the line, with the position resolved fresh via `posAtDOM`. The byte-round-trip suites (`stamp-hide`, the inline-reveal round-trips) still pass. Keep new CM6 mutations to that pattern.
5. **The pin/transclusion click-through is browser-only.** `posAtDOM(wrap)` closures inside a mounted CM widget cannot be headlessly tested; the pure rewrite helpers were tested against a real `EditorView` with explicit positions, and the React control wiring via component tests. Image-type snapshots need a real canvas. These are Grant's :3000 checks.
6. **ai-helper regen.** None of this session's work added a new lib TYPE that the ai-helper scans (the new files are components + helpers consumed internally), so no `public/ai-helper/*` regen was needed. If a future embed phase adds a new exported lib type, regenerate (`build-ai-helper.mjs` + `check-ai-helper.mjs`) or the build gate fails.
