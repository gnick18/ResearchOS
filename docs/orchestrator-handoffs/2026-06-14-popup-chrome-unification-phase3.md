## LANDMINES / lessons (hard-won this session)
- **Shared main checkout is HEAVILY dirty + many lanes commit/merge concurrently.** Editing files directly in main and waiting to commit = your uncommitted edit gets RESET out from under you (happened to the white-input fix: `ca1fec882` silently dropped it; re-applied atomically as `87f775d87`). For tiny on-main fixes: edit + `git add` + `git commit` in ONE atomic step. For anything bigger: build in a worktree, merge atomically (`git merge --no-ff`, single command). [[feedback_integrate_from_worktree]] [[feedback_shared_manifest_race]]
- **Merging into the dirty main with a file another lane has uncommitted:** stash JUST that file (`git stash push -- <file>`), merge, `git stash pop` ΓÇË restores their uncommitted work without sweeping it into your commit. Used for `providers.tsx` (their `queueMicrotask` fix vs my dev bypass, 500 lines apart, clean).
- **icon-guard pre-commit scans the WHOLE src tree** vs `frontend/icon-svg-baseline.json`; a new file with an un-baselined inline `<svg>` blocks EVERY lane's commits. Dev throwaways go in `EXCLUDED_PREFIXES` (`update-icon-baseline.mjs`).
- **Render gate visual half needs a live renderer.** Orchestrator can't drive Chrome solo here (extension not connected to the MCP); Grant runs the visual pass on his `:3000` via the dev page or the extension. Code-level adversarial verifier (independent agent) catches structural drift first.
